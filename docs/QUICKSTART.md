# Elementor MCP — Quick Start

A 3-step recipe for spinning up a new WordPress + Elementor build that Claude Code can drive end-to-end. Works for **Local-by-Flywheel** and **live hosts** alike.

---

## What this gives you

After ~3 minutes of setup, Claude Code can:

- Set Elementor global colors and typography (your design tokens)
- Create pages, set them as the homepage
- Build sections — containers, headings, images, buttons, tabs, forms, custom HTML
- Edit existing pages, headers, footers
- Read and modify any Elementor element by ID

The MCP exposes ~75 Elementor tools. Claude loads schemas lazily as needed.

---

## Prerequisites

**On the WordPress site** *(install before running setup):*

| Plugin | Required? | Notes |
|---|---|---|
| Elementor (free) | **Yes** | The page builder itself |
| Hello Elementor (theme) | Strongly recommended | Blank canvas — other themes inject CSS that fights Elementor |
| Header Footer Elementor (Brainstorm Force / UAE) | If building custom header/footer | Theme Builder is Pro; this fills the gap |
| Fluent Forms / WPForms / CF7 | If you want a real contact form | Form widget is Pro |
| Essential Addons Lite | Optional | Adds free Post Grid widget |

*The setup script will warn you if Elementor or Hello Elementor are missing.*

**On your machine:**

- macOS or Linux (script is bash)
- `curl`, `python3`, `unzip`, `zip` (all preinstalled on macOS)
- Claude Code (CLI or desktop app)
- For Local sites: Local-by-Flywheel installed and the site **started**

## Step 1 — Run the setup wizard

From inside your **project directory** (where you want the `.mcp.json` to live — typically the folder where your design files are):

```bash
bash ~/.claude/scripts/setup-elementor-mcp.sh
```

The wizard walks you through 7 steps:

1. **Local vs live** — pick (1) for `~/Local Sites/...` or (2) for any live URL
2. **Site URL** — for Local it auto-detects `<site-name>.local`; for live you paste the full URL
3. **Connectivity probe** — confirms the WP REST API is reachable
4. **Auth** — you create an Application Password in WP Admin and paste it
   > **Critical gotcha:** the password's *name/label* is just a label. The username is your actual WP login (e.g. `admin`, not "ClaudeMCP"). The wizard will list public users if auth fails so you can find the right slug.
5. **Plugin baseline** — confirms Elementor is active and warns if Hello Elementor isn't the theme
6. **Plugin install** — handles the GitHub-only zips:
   - On Local: installs both via the bundled WP-CLI (auto-finds the per-site MySQL socket — this is the part that's painful by hand)
   - On live: prints the two `.zip` paths and waits while you upload them via WP Admin → Plugins → Add New → Upload Plugin
7. **`.mcp.json` written** to the current directory

When it prints `✓ Setup complete`, you're 90% done.

---

## Step 2 — Restart Claude Code

`.mcp.json` is only read at startup. So:

1. **Quit** Claude Code (Cmd-Q for the app; Ctrl-C if running the CLI)
2. **Reopen it inside this project directory**
3. Claude Code will detect the new `.mcp.json` and ask you to **approve the `elementor` MCP server** — approve it.

---

## Step 3 — Tell Claude to build

Just describe what you want. Examples:

- *"Set up global colors from this CSS file and create a homepage"*
- *"Build the hero section using the design in this project HTML mockup"*
- *"Add a contact form section with a Fluent Forms shortcode"*
- *"Show me the structure of page ID 11 and tell me what's wrong with the spacing"*

Claude will auto-load the **`elementor-mcp` skill** (at `~/.claude/skills/elementor-mcp/SKILL.md`), which encodes all the conventions:

- Flat parameter shape (don't wrap in `settings: {}`)
- `flex_direction` / `flex_justify_content` / `flex_align_items` key names
- `typography_typography: "custom"` to enable typography overrides
- Italic emphasis via inline `<em>` in headings
- When to use HTML widget vs native widgets (card grids, complex hover effects, content inside Tabs)
- Header/footer flow via Header Footer Elementor (Free) vs Theme Builder (Pro)

## Common scenarios

### "I'm starting a new client site from scratch"

1. Spin up a new Local site, install plugins, switch to Hello Elementor
2. Run `setup-elementor-mcp.sh` from the project folder where your design lives
3. Restart Claude Code
4. Tell it "use the Elementor MCP to build a page based on `<your-design-file>`"

### "I have an existing live WordPress site and want to use Claude on it"

1. From WP Admin: ensure Elementor + Hello Elementor + (optional) HFE/Fluent Forms are installed
2. Create an Application Password in your profile
3. Run the setup script, choose option 2 (live host)
4. Upload the two MCP plugin zips via WP Admin when the script prompts you
5. Restart Claude Code, approve the MCP

### "I want to switch the MCP between two sites"

Each project directory can have its own `.mcp.json`. Just `cd` to a different project before running the setup script — it writes to the current directory.

### "Setup script failed — what now?"

Common causes:

- **Auth fails** — username is wrong. The script will list public users; pick the right slug.
- **`Could not find MySQL socket`** — Local site isn't running. Open Local, click Start Site, re-run.
- **Live host returns 403 from `/wp-json/`** — some hosts block non-browser User-Agents. Add the WP-CLI's IP to your security plugin's allowlist or temporarily disable the firewall.
- **MCP namespace doesn't appear after install** — the elementor-mcp plugin requires Elementor v3.20+. Check the elementor-mcp activation row in WP Admin → Plugins for any error message.

## What this approach is good at

- **Section-by-section visual builds** of marketing pages — heroes, intros, listings grids, neighborhoods, stats, journal, contact
- **Translating a designed HTML mockup** into Elementor — Claude reads your reference HTML/CSS and emits widget calls
- **Iterating quickly** — "make the hero headline 20% smaller", "swap the burgundy for a navy", "remove the stats band"
- **Bulk edits across pages** — "find all containers with padding 200px and change to 160px"

## What it isn't

- **Pixel-perfect HTML→Elementor compilation** — Elementor's flexbox container model is the ceiling; some layouts you'd write in 10 lines of CSS need 5 nested containers in Elementor
- **A replacement for Elementor's visual editor** — for visual fine-tuning (drag handles, hover preview), the editor is still the best tool
- **A way around Elementor Pro features** — Posts widget, Form widget, Theme Builder, Loop Grid, Sticky/Motion effects all remain Pro. The skill documents free workarounds.
